# Athoo Documentation

- `architecture/` — current technical architecture and provider contracts, including map and communication-provider switching.
- `runbooks/` — deployment, incident, rollback, device and launch procedures.
- `policies/` — operational and data-handling policies.
- `qa/` — release checklists and evidence templates.
- `archive/development-history/` — historical phase reports, audits and superseded certification snapshots.

Only current operational documents should be kept outside the archive. New phase reports belong in `archive/development-history/`; durable architecture and operational guidance belongs in the relevant current folder.
