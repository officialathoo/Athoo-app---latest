# Closed-Beta Feedback Triage

Every report must include app version, platform/build number, user role, environment, timestamp, affected booking/user ID when applicable, reproducible steps, expected result, actual result, and screenshots with private data redacted.

Priority:
- **P0:** security, privacy, financial integrity, data loss, or total outage. Stop beta and invoke incident response.
- **P1:** booking, login, provider acceptance, PIN, chat, or admin workflow blocked.
- **P2:** degraded workflow with a safe workaround.
- **P3:** cosmetic, wording, or low-impact usability issue.

A defect is closed only after a linked code change, regression test where practical, staging verification, and confirmation on the affected platform.
